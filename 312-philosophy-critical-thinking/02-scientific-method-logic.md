## Индукция и дедукция

### Дедукция (от общего к частному)

```
Все люди смертны.      ← общее правило
Сократ — человек.      ← частный случай
Сократ смертен.        ← вывод (логически необходимый)
```

**Дедуктивный аргумент валиден:** если посылки истинны, вывод **обязательно** истинен.

### Индукция (от частного к общему)

```
Лебедь #1 — белый.
Лебедь #2 — белый.
...
Лебедь #10000 — белый.
Все лебеди — белые.    ← вывод (вероятностный, не гарантированный)
```

**Индуктивный аргумент:** посылки поддерживают вывод, но не гарантируют его (чёрные лебеди существуют).

| | Дедукция | Индукция |
|---|----------|----------|
| **Вывод** | Необходимый | Вероятностный |
| **Ошибка** | Невалидность | Слабая поддержка |
| **Пример** | Математика | Наука, ML |

## Абдукция (лучшее объяснение)

```
Наблюдение: Трава мокрая.
Правило: Если шёл дождь, трава мокрая.
Вывод: Вероятно, шёл дождь. (лучшее объяснение)
```

**Абдукция** — вывод к наилучшему объяснению. Используется в диагностике, детективах, AI.

## Фальсификация (Karl Popper)

### Критерий демаркации

**Научная теория** = теория, которую можно **опровергнуть** экспериментально.

| Теория | Фальсифицируема? | Научна? |
|--------|-------------------|---------|
| «Все тела притягиваются» | Да (достаточно одного негравитирующего тела) | Да |
| «Все люди бессмертны» | Нет (невозможно доказать обратное) | Нет |
| «Психонализ объясняет всё» | Нет (любое поведение подтверждает) | Нет (по Попперу) |

### Импликация для инженерии

- **Тестирование** не доказывает отсутствие багов (Popper: verification impossible)
- Один проваленный тест = фальсификация (опровержение)
- TDD: сначала пишем опровергающий тест, потом код

## Бритва Оккамы (Occam's Razor)

> "Entities should not be multiplied without necessity."

**Современная формулировка:** при прочих равных, простое объяснение лучше сложного.

### В разработке

```
❌ Архитектура с микросервисами для TODO-приложения
✅ Монолит (пока не нужна масштабируемость)
```

**Важно:** Бритва Оккамы — **эвристика**, не закон. Сложное решение может быть правильным, если простое не работает.

### Hicks's Razor (для идей)

> "The simpler explanation is more likely to be true, provided it explains the facts."

### Hitchens's Razor

> "What can be asserted without evidence can be dismissed without evidence."

## Эвристики и когнитивные искажения

### Эвристики (быстрые, но неточные)

| Эвристика | Описание | Ошибка |
|-----------|----------|--------|
| **Availability** | Оцениваем вероятность по лёгкости воспоминания | «Авиакатастрофы страшнее, чем автокатастрофы» (хотя авто убивают больше) |
| **Representativeness** | Судим по похожести на стереотип | «Он в очках и читает книги → точно программист» |
| **Anchoring** | Первая информация задаёт якорь | «Скидка 50% от 1000р → выгодно» (хотя товар стоит 300р) |

### Когнитивные искажения (Cognitive Biases)

| Искажение | Описание | В разработке |
|-----------|----------|--------------|
| **Dunning-Kruger** | Неопытные переоценивают себя, эксперты недооценивают | Junior уверен в решении, Senior сомневается |
| **Sunk Cost** | Продолжаем, потому что уже вложились | «Мы уже 2 месяца пишем этот микросервис, нельзя бросить» |
| **Planning Fallacy** | Систематически недооцениваем время | «Это займёт 2 дня» → занимает 2 недели |
| **Hindsight Bias** | «Я же говорил» после события | Post-mortem без blame — борьба с этим |
| **IKEA Effect** | Ценим то, что сделали сами | «Моя архитектура лучше», потому что я её придумал |
