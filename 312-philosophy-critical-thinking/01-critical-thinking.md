# Основы критического мышления

---

## Что такое критическое мышление

> "Critical thinking is the systematic evaluation of arguments based on reason and evidence."

Это не «критика в негативном смысле», а **дисциплинированное мышление**, которое:
- Анализирует аргументы
- Оценивает доказательства
- Выявляет предубеждения
- Делает обоснованные выводы

## Логические ошибки (Logical Fallacies)

### Формальные ошибки

| Ошибка | Пример | Почему неверно |
|--------|--------|----------------|
| **Affirming the consequent** | «Если идёт дождь, трава мокрая. Трава мокрая → значит, шёл дождь» | Может быть роса, полив |
| **Denying the antecedent** | «Если это собака, то она лает. Это не собака → значит, не лает» | Кошка тоже может шипеть |
| **Illicit major/minor** | «Все кошки — млекопитающие. Все собаки — не кошки. Значит, собаки — не млекопитающие» | Нарушение распределения терминов |

### Неформальные ошибки

| Ошибка | Пример |
|--------|--------|
| **Ad Hominem** | «Ты не можешь говорить о климате, ты же не учёный» (атака на человека, не на аргумент) |
| **Straw Man** | «Ты за снижение налогов? Значит, хочешь закрыть все школы?» (искажение позиции) |
| **False Dilemma** | «Или ты с нами, или ты враг» (только два варианта, хотя есть третий) |
| **Slippery Slope** | «Если разрешим однополые браки, люди начнут жениться на животных» (цепочка без доказательств) |
| **Appeal to Authority** | «Эйнштейн верил в Бога → значит, Бог есть» (авторитет в одной области не переносится) |
| **Appeal to Nature** | «Это натуральное → значит, полезное» (природа не всегда полезна) |
| **Confirmation Bias** | Искать только то, что подтверждает твою теорию |
| **Survivorship Bias** | «Все успешные стартапы работали 80 часов. → Работай 80 часов» (тех, кто прогорел, уже не видно) |

### Анекдот про инженера и логику

> Инженер и математик сидят в поезде. За окном — чёрная овца.
> Математик: «В Шотландии все овцы чёрные».
> Инженер: «В Шотландии есть как минимум одна овца, одна сторона которой — чёрная».

## Аргументация

### Структура аргумента (Toulmin Model)

```
Claim     — утверждение ("SQL лучше NoSQL для этого проекта")
Grounds   — данные / evidence ("Нам нужны ACID-транзакции")
Warrant   — связь между claim и grounds ("ACID требует реляционную БД")
Qualifier — степень уверенности ("Вероятно", "В большинстве случаев")
Rebuttal  — контр-аргументы ("Если шардинг, то..."
Backing   — поддержка warrant ("Исследование CAP theorem")
```

### Карта аргумента (Argument Mapping)

```
                       ┌─── Claim ───┐
                      /       |       \
                     ▼        ▼        ▼
                  Reason 1  Reason 2  Objection
                     │        │         │
                     ▼        ▼         ▼
                  Evidence Evidence  Alternative
                  /  Data    / Data
```

## Фаллибилизм (Fallibilism)

**Фаллибилизм** — философская позиция, что любое знание может оказаться ошибочным.

| Мысль | Суть |
|-------|------|
| **Поппер** | Наука не доказывает истину, а опровергает ложь. Теория хороша, пока не опровергнута. |
| **Сократ** | «Я знаю, что ничего не знаю» — осознание границ знания |
| **Рассел** | «Проблема мира в том, что глупцы самоуверенны, а умные полны сомнений» |

### Применение в разработке

- Ваше архитектурное решение — **гипотеза**, которая может быть неверной
- Code review — фальсификация (попытка опровергнуть)
- ADR — документирование rationale на случай, если решение окажется неверным
- «It depends» — правильный ответ на 90% вопросов

## Научный скептицизм

### Принципы

1. **Extraordinary claims require extraordinary evidence** (Carl Sagan)
2. **Burden of proof** лежит на утверждающем
3. **Falsifiability** — утверждение должно быть опровергаемо, чтобы быть научным
4. **Occam's Razor** — не множь сущности без необходимости

### Sagan's Baloney Detection Kit

```
1. Где доказательства? (а не просто истории)
2. Можно ли проверить независимо?
3. Кто проводил исследование? (нет конфликта интересов?)
4. Есть ли альтернативные объяснения?
5. Используется ли фаллибилизм? (автор признаёт, что может ошибаться?)
6. Корреляция ≠ причинность?
7. Это наука или псевдонаука? (falsifiable?)
```
