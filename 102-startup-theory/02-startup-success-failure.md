## Истории успеха

### Airbnb (2008 – н.в.)

| Параметр | Значение |
|---|---|
| **Год основания** | 2008 |
| **Основатели** | Brian Chesky, Joe Gebbia, Nathan Blecharczyk |
| **Идея** | Сдавать надувные матрасы (airbeds) в своей квартире во время конференции в SF |
| **Текущая капитализация** | ~$80-100B (2024) |
| **Ключ к успеху** | PMF в нише, виральный growth через Craigslist интеграцию, сильный community бренд |

**Причины успеха:**
1. **Эффективная bootstrap-стратегия** — продажа Obama O's cereal для финансирования (2008)
2. **Product-Market Fit** — путешественники хотели дешевле отелей, хозяева — заработать
3. **Growth hack** — автоматический постинг на Craigslist из Airbnb
4. **Доверие** — система отзывов, верификация, страхование хозяев ($1M guarantee)

### Slack (2013 – н.в.)

| Параметр | Значение |
|---|---|
| **Год основания** | 2013 (как продукт) |
| **Основатель** | Stewart Butterfield |
| **Идея** | Internal tool для игры Glitch → корпоративный мессенджер |
| **Текущая капитализация** | Куплен Salesforce за $27.7B (2021) |
| **Ключ к успеху** | Pivot (zoom-out): внутренний инструмент → платформа |

**Причины успеха:**
1. **Pivot вовремя** — Glitch не взлетел, но внутренний чат оказался нужнее
2. **Product-led growth** — пользователи сами приносили Slack в свои компании
3. **Интеграции** — открытая платформа (API, боты, коннекторы)
4. **UX** — на порядок лучше конкурентов (HipChat, Skype)

## Истории провала

### Webvan (1999 – 2001)

| Параметр | Значение |
|---|---|
| **Год основания/краха** | 1999 → 2001 |
| **Идея** | Доставка продуктов онлайн |
| **Привлечено** | ~$800M (включая IPO) |
| **Сожжено** | ~$1.2B |
| **Ключевая причина** | Слишком рано, слишком капиталоёмко |

**Причины провала:**
1. **Overfunding / overscaling** — построили роботизированные склады за $40M каждый до PMF
2. **Нет PMF** — рынок e-grocery был не готов (низкое проникновение интернета)
3. **Unit economics не работали** — LTV < CAC, доставка дешевле себестоимости
4. **Слишком быстро** — $375M IPO в 1999, банкротство в 2001

**Урок:** Масштабироваться нужно после PMF, а не до.

### Pets.com (1998 – 2000)

| Параметр | Значение |
|---|---|
| **Год основания/краха** | 1998 → 2000 |
| **Идея** | Онлайн-продажа товаров для животных |
| **Привлечено** | ~$300M |
| **Сожжено** | ~$300M |
| **Ключевая причина** | Unit economics не работали |

**Причины провала:**
1. **Negative unit economics** — доставка корма для животных стоила дороже самого корма
2. **Массовый маркетинг без удержания** — $20M на рекламу (с баллонами на Macy's Thanksgiving Parade)
3. **Нет LTV** — разовая покупка, низкая частота, слабая лояльность
4. **Cash burn** — ~$300M сожжено за 18 месяцев

**Урок:** Без positive unit economics никакой маркетинг не спасёт.

### Theranos (2003 – 2018)

| Параметр | Значение |
|---|---|
| **Год основания/краха** | 2003 → 2018 (крах) |
| **Основатель** | Elizabeth Holmes |
| **Идея** | Анализ крови из одной капли |
| **Привлечено** | ~$700M |
| **Оценка (пик)** | $9B (2015) |
| **Ключевая причина** | Fraud — технология не работала |

**Причины провала:**
1. **Fraud** — Holmes сознательно врала о возможностях технологии
2. **Культура секретности** — не допускали независимую верификацию
3. **Board of directors** — сильные имена (George Shultz, Henry Kissinger), но никто не разбирался в технологии
4. **Игнорирование whistleblower'ов** — увольнение сотрудников, указывавших на проблемы

**Урок:** Технология должна работать. Сильные связи и деньги не заменяют научного метода.

### Quibi (2018 – 2020)

| Параметр | Значение |
|---|---|
| **Год основания/краха** | 2018 → 2020 |
| **Основатель** | Jeffrey Katzenberg |
| **Идея** | Короткие видео (до 10 мин) для мобильных — Hollywood quality |
| **Привлечено** | $1.75B |
| **Сожжено** | $1.75B |
| **Ключевая причина** | Отсутствие PMF |

**Причины провала:**
1. **Нет PMF** — люди не хотели платить $5/мес за short-form видео (TikTok — бесплатно)
2. **Неверное предположение** — Hollywood контент vs UGC. Пользователи хотели UGC, а не произведённое видео
3. **Пандемия** — запуск в апреле 2020 (люди дома, не в транспорте, где "короткие" видео нужны)
4. **Product-market timing** — если бы не COVID, могло бы продержаться дольше

**Урок:** Если конкуренты (TikTok, YouTube Shorts, Instagram Reels) дают то же самое бесплатно — нужна очень веская причина платить.

### Juicero (2013 – 2017)

| Параметр | Значение |
|---|---|
| **Год основания/краха** | 2013 → 2017 |
| **Идея** | Wi-Fi соковыжималка с пакетами |
| **Привлечено** | $120M |
| **Ключевая причина** | Over-engineering, нет PMF по цене |

**Причины провала:**
1. **Over-engineering** — соковыжималка за $700 + $5-7 за пакет сока
2. **Product failure** — выяснилось, что пакеты сока можно выжать руками
3. **Цена** — $700 за соковыжималку при рынке $50-200
4. **Экологичность** — одноразовые пластиковые пакеты

**Урок:** Технология ради технологии не создаёт ценности. Проблема должна быть реальной.

### WeWork (2010 – 2019*)

| Параметр | Значение |
|---|---|
| **Год основания** | 2010 |
| **Основатель** | Adam Neumann |
| **Идея** | Coworking space / community |
| **Пиковая оценка** | $47B (2019) |
| **Текущая оценка** | ~$300M-1B (2024) |
| **Ключевая причина** | Overvaluation + corporate governance failure |

**Причины провала (провал IPO 2019):**
1. **Corporate governance** — Neumann контролировал компанию через multi-class shares, конфликт интересов
2. **Финансовые махинации** — подавали финансовые показатели, исключая расходы ("Community Adjusted EBITDA" не включал аренду)
3. **Overvaluation** — $47B за бизнес по сдаче офисов в субаренду (WeWork's core business — lease arbitrage)
4. **Убытки** — $1.9B loss на $2.9B revenue (2018)

**Урок:** Если бизнес — аренда столов, он не стоит как tech company. Прозрачность и governance — критичны.

### Google+ (2011 – 2019)

| Параметр | Значение |
|---|---|
| **Год запуска/закрытия** | 2011 → 2019 |
| **Компания** |
**Привлечено/сожжено** | ~$500M+ (оценка) |
| **Ключевая причина** | Product failure, отсутствие виральности |

**Причины провала:**
1. **No viral loop** — Google заставлял пользователей регистрироваться (YouTube, Gmail интеграция), но органического роста не было
2. **Product strategy** — "Ghost town" — пользователи регистрировались, но не возвращались
3. **Social graph ≠ interest graph** — Google circles (круги) были умнее, чем у Facebook, но сложнее для пользователя
4. **Поздний запуск** — Facebook уже захватил social graph пользователей

**Урок:** Заставить зарегистрироваться можно, заставить остаться — нельзя. Виральность и engagement не купить.

## Главные выводы

### Топ-5 причин успеха:
1. **PMF до масштабирования** — Airbnb, Slack, Instagram
2. **Pivot вовремя** — Slack (game→chat), Instagram (check-in→photo), Netflix (DVD→streaming)
3. **Unit economics работают** — LTV/CAC > 3, NRR > 100%
4. **Product-led growth** — пользователи приводят пользователей (Slack, Stripe, TikTok)
5. **Тайминг** — правильное время для рынка (TikTok — 4G/5G, Stripe — SaaS boom)

### Топ-5 причин провала:
1. **Нет PMF** — Quibi, Google+
2. **Плохие unit economics** — Pets.com, Webvan
3. **Fraud / Governance failure** — Theranos, WeWork
4. **Over-engineering / Overfunding** — Juicero, Webvan
5. **Нет viral loop** — Google+
